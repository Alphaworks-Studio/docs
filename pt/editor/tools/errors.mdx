Detecção de erros analisa seu modelo em busca de problemas de geometria que podem causar falhas de impressão ou artefatos visuais.

## Quando usar esta ferramenta

- Antes do reparo, para entender o que há de errado com seu modelo
- Após o reparo, para verificar se todos os problemas foram resolvidos
- Quando seu modelo parece errado ou tem lacunas inesperadas

## O que detecta

O painel abre com um status **Estanque** (Sim/Não) e, em seguida, lista os problemas agrupados por gravidade.

Antes do reparo, os problemas são derivados da análise do seu modelo:

| Problema | O que significa |
|-------------|----------------|
| **Malha não estanque** | A malha tem arestas de contorno abertas e não está vedada |
| **Múltiplas shells** | O modelo contém mais de uma shell separada — mostra quantas foram detectadas |
| **Problemas detectados** | Outros problemas de geometria encontrados durante a análise — mostra uma contagem |

Após o reparo, o painel lista os problemas retornados pelo reparo.

## Resultados

- **Contagem total de problemas** exibida como um selo ao lado do título do painel
- **Agrupados por gravidade** — os problemas são listados sob os cabeçalhos Crítico, Aviso e Informação, cada um com uma contagem
- **Status Corrigido / Não corrigido** — após um reparo, cada problema mostra se foi resolvido

## Dicas

- **Arestas não-manifold** são o problema mais comum — significam que as superfícies não formam um sólido adequado
- Execute isso **antes do reparo** para entender a extensão dos problemas
- Após o reparo, a contagem de erros deve ser **zero ou próxima de zero**
- Se erros persistirem após o reparo, tente um [preset de tipo de modelo](/repair/model-types) diferente
